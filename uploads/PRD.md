## 1. 本质与目标

### 1.1 核心问题定义（第一性原理分析）

**设计思考**：传统刷题平台存在三大痛点：
1. **效率痛点**：用户难以快速找到适合自己水平的题目，刷题效率低
2. **辅导痛点**：缺乏即时解题思路引导，用户遇到难题时只能被动等待
3. **沉淀痛点**：错题遗忘、知识点散乱，无法形成系统化学习闭环

AI刷题应用的核心价值在于：通过AI智能分析弥补"人-题"之间的认知鸿沟，实现"找题-答题-分析-沉淀"的全链路智能化闭环。

### 1.2 商业目标

| 目标类型 | 具体指标 |
|---------|---------|
| **用户价值** | 提升用户刷题效率30%以上，减少无效刷题时间 |
| **产品定位** | 智能化刷题工具 + AI学习助手 |
| **长期愿景** | 打造"AI驱动"的个人学习知识图谱平台 |

### 1.3 成功指标（North Star Metric）

| 指标 | 目标值 | 衡量周期 |
|------|--------|---------|
| **日活跃用户（DAU）** | > 100 | MVP上线后30天 |
| **人均刷题数** | > 5题/天 | MVP上线后30天 |
| **AI分析使用率** | > 60% | MVP上线后30天 |
| **错题复习转化率** | > 40% | MVP上线后30天 |

## 3. 场景全景

### 3.1 正常流（用户旅程图）

```
用户注册登录 → 浏览题库 → 选择题目 → 开始答题 → 提交答案
    ↓
查看结果 → AI题目分析 → 查看解析 → 标记状态 → 下一题
    ↓
错题自动归档 → 学习计划提醒 → 复习错题 → AI错题分析 → 知识点归纳
```

### 3.2 用户旅程详细流程

#### 主刷题流程

| 步骤 | 用户行为 | 系统响应 | 数据变更 |
|------|---------|---------|---------|
| 1 | 用户登录 | 验证邮箱密码，返回用户信息 | - |
| 2 | 浏览题库列表 | 返回题目列表（含筛选条件） | - |
| 3 | 选择题目 | 返回题目详情页 | - |
| 4 | 开始答题 | 记录答题开始时间 | QuestionLog.status=进行中 |
| 5 | 提交答案 | 判断正确性，返回结果 | QuestionLog更新 |
| 6 | 查看AI分析 | 调用AI生成分析内容 | AIConversation新增 |
| 7 | 查看解析 | 返回标准答案+知识点 | - |
| 8 | 标记状态 | 更新题目状态 | QuestionLog.status更新 |

### 3.3 异常流与边缘流

#### 网络异常场景

| 场景 | 触发条件 | 系统行为 | 用户提示 |
|------|---------|---------|---------|
| **弱网提交** | 网络延迟>5秒 | 本地缓存答案，后台重试 | "提交中，请稍候..." |
| **断网答题** | 网络完全中断 | 本地存储答题进度 | "网络异常，答案已本地保存" |
| **重连恢复** | 网络恢复后 | 自动同步本地数据 | "数据已同步" |

#### AI服务异常场景

| 场景 | 触发条件 | 系统行为 | 用户提示 |
|------|---------|---------|---------|
| **AI超时** | API响应>10秒 | 终止请求，返回降级提示 | "AI服务繁忙，请稍后重试" |
| **AI不可用** | 服务返回503 | 记录日志，降级处理 | "AI分析暂时不可用" |
| **AI次数耗尽** | 用户已达每日上限 | 拒绝请求，显示剩余次数 | "今日AI分析次数已用完，明天再来" |

#### 数据异常场景

| 场景 | 触发条件 | 系统行为 | 用户提示 |
|------|---------|---------|---------|
| **题目数据为空** | 题库无数据 | 显示引导提示 | "暂无题目，请联系管理员添加" |
| **错题数据丢失** | 同步失败 | 本地缓存，后台重试 | "数据同步中..." |
| **收藏失败** | 数据库异常 | 提示重试 | "收藏失败，请重试" |

#### 并发冲突场景

| 场景 | 触发条件 | 系统行为 | 用户提示 |
|------|---------|---------|---------|
| **重复提交答案** | 短时间多次提交 | 忽略后续请求 | "答案已提交" |
| **并发答题** | 多设备同时答题 | 取最新设备状态 | "已同步最新答题状态" |

## 5. 详细需求（FR）

### FR-01：用户登录

| 维度 | 内容 |
|------|------|
| **输入** | 邮箱地址、密码、确认密码（注册时） |
| **处理逻辑** | 1. 验证邮箱格式有效性<br>2. 检查邮箱是否已注册<br>3. 密码加密存储（bcrypt）<br>4. 生成JWT Token |
| **输出** | 登录成功返回用户信息+Token；失败返回错误原因 |
| **数据规则** | 邵箱唯一；密码长度≥8位；密码找回发送邮件链接 |
| **异常处理** | 邵箱格式错误提示"邮箱格式不正确"；密码错误提示"密码错误，请重试" |

### FR-02：题目浏览

| 维度 | 内容 |
|------|------|
| **输入** | 筛选条件：分类、标签、难度、状态、关键词 |
| **处理逻辑** | 1. 解析筛选参数<br>2. 查询题库表<br>3. 分页返回结果 |
| **输出** | 题目列表（含：id、标题、分类、难度、状态、标签） |
| **数据规则** | 默认每页20题；支持排序（难度/时间/正确率） |
| **异常处理** | 筛选结果为空时显示"暂无符合条件的题目" |

### FR-03：题目答题

| 维度 | 内容 |
|------|------|
| **输入** | 题目ID、用户选择的答案选项 |
| **处理逻辑** | 1. 验证题目存在<br>2. 判断答案正确性<br>3. 更新QuestionLog状态<br>4. 更新题目状态流转 |
| **输出** | 答题结果（正确/错误）+ 标准答案 + AI分析入口 |
| **数据规则** | 单选题唯一答案；多选题需全部匹配；状态按五态流转 |
| **异常处理** | 弱网提交失败提示重试；重复提交忽略后续请求 |

### FR-04：AI题目分析

| 维度 | 内容 |
|------|------|
| **输入** | 题目ID、题目内容、用户答案（可选） |
| **处理逻辑** | 1. 检查用户AI次数余额<br>2. 调用兼容OpenAI格式API<br>3. 解析AI返回内容<br>4. 存储对话记录 |
| **输出** | AI分析结果：题目内容分析、解题思路、知识点标注 |
| **数据规则** | 每日分析次数上限（默认10次）；响应超时10秒 |
| **异常处理** | 超时提示"AI服务繁忙"；次数耗尽提示"今日次数已用完" |

### FR-05：AI错题分析

| 维度 | 内容 |
|------|------|
| **输入** | 用户错题记录、题目内容、用户错误答案 |
| **处理逻辑** | 1. 调用AI分析错题原因<br>2. 归纳错误类型（概念错误/计算错误/理解偏差）<br>3. 推荐相关知识点 |
| **输出** | 错题原因分析、错误类型、改进建议 |
| **数据规则** | 消耗AI次数；支持多轮追问 |
| **异常处理** | AI失败时提示"请手动分析错题原因" |

### FR-06：AI多轮对话

| 维度 | 内容 |
|------|------|
| **输入** | 用户提问内容、历史对话上下文 |
| **处理逻辑** | 1. 构建对话上下文（最近5轮）<br>2. 调用AI生成回复<br>3. 存储对话记录 |
| **输出** | AI回复内容、追问入口 |
| **数据规则** | 上下文保留最近5轮；每次对话消耗AI次数 |
| **异常处理** | 超时降级；对话记录存储失败时队列缓存 |

### FR-07：题库管理（管理端）

| 维度 | 内容 |
|------|------|
| **输入** | 题目标题、内容、分类、难度、标签、答案、解析 |
| **处理逻辑** | 1. 验证数据完整性<br>2. 检查分类/标签有效性<br>3. 存储题目数据<br>4. 记录操作日志 |
| **输出** | 操作成功/失败结果 |
| **数据规则** | 标题必填；内容必填；答案必填；难度枚举（简单/中等/困难） |
| **异常处理** | 操作失败提示具体原因；记录异常日志 |

### FR-08：批量导入导出（管理端）

| 维度 | 内容 |
|------|------|
| **输入** | Excel/JSON文件 |
| **处理逻辑** | 1. 解析文件格式<br>2. 验证数据字段<br>3. 批量插入数据库<br>4. 返回导入结果统计 |
| **输出** | 导入成功数、失败数、失败原因列表 |
| **数据规则** | Excel字段映射；JSON Schema校验 |
| **异常处理** | 格式错误提示"文件格式不正确，请检查模板"；字段缺失提示具体缺失项 |

### FR-09：运营数据统计（管理端）

| 维度 | 内容 |
|------|------|
| **输入** | 时间范围筛选 |
| **处理逻辑** | 聚合统计：DAU、题目完成数、AI调用次数、新注册用户数 |
| **输出** | 统计报表（图表+数据表） |
| **数据规则** | 默认最近7天；支持自定义范围 |
| **异常处理** | 数据延迟时显示上次缓存结果 |

## 7. 数据与规则（BR）

### 7.1 实体关系图

```
User (用户)
  │
  ├── 1:N → QuestionLog (答题记录)
  │
  ├── 1:N → Note (笔记)
  │
  ├── 1:N → Favorite (收藏)
  │
  ├── 1:N → AIConversation (AI对话)
  │
  └── 1:1 → StudyPlan (学习计划)

Question (题目)
  │
  ├── 1:N → QuestionLog (答题记录)
  │
  ├── 1:N → Comment (评论)
  │
  └── 1:N → Note (笔记)

Category (分类)
  │
  └── 1:N → Question (题目)
```

### 7.2 核心实体字段定义

#### User（用户表）

| 字段 | 类型 | 约束 | 说明 |

| avatar | VARCHAR(500) | - | 头像URL |
| role | ENUM | NOT NULL | 角色：user/admin |
| ai_daily_limit | INT | DEFAULT 10 | AI每日次数上限 |
| ai_used_today | INT | DEFAULT 0 | 今日已用次数 |
| created_at | TIMESTAMP | NOT NULL | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL | 更新时间 |

#### Question（题目表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 主键 |
| title | VARCHAR(255) | NOT NULL | 题目标题 |
| content | TEXT | NOT NULL | 题目内容 |
| category_id | UUID | FK | 分类ID |
| difficulty | ENUM | NOT NULL | 难度：easy/medium/hard |

### 7.3 题目状态机（五态流转）

```
                    ┌─────────────────────────────────────┐
                    │                                     │
                    ▼                                     │
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│   未答   │───▶│  进行中  │───▶│   已答   │───▶│ 需复习   │───▶│  已掌握  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                    │
                                                    │ 错题重新答题后
                                                    ▼
                                               ┌──────────┐
                                               │   已答   │
                                               └──────────┘
```

**状态流转规则**：
- **未答 → 进行中**：用户打开题目详情页
- **进行中 → 已答**：用户提交答案
- **已答 → 需复习**：答案错误，自动归入错题
- **需复习 → 已答**：用户重新答题
- **已答 → 已掌握**：答案正确，且复习次数≥2次
- **已掌握 → 需复习**：用户手动标记需复习

### 7.4 数据生命周期管理

| 实体 | 创建规则 | 更新规则 | 归档规则 | 删除规则 |
|------|---------|---------|---------|---------|
| **User** | 注册时创建 | 资料修改时更新 | 90天不活跃归档 | 用户申请删除或违规封禁 |
| **Question** | 管理员添加 | 管理员编辑 | 过期题目归档 | 管理员删除 |
| **QuestionLog** | 答题时创建 | 答题/复习时更新 | 保留180天 | 用户删除账号时清理 |
| **Note** | 用户创建 | 用户编辑 | 保留180天 | 用户删除 |
| **AIConversation** | AI对话时创建 | 对话追加 | 保留30天 | 自动清理 |

## 9. 决策记录

### DR-01：选择纯文字题而非编程题

| 维度 | 内容 |
|------|------|
| **背景** | 刷题应用类型选择 |
| **选项** | A. 编程题（代码执行沙箱）<br>B. 纯文字题（选择题/判断题） |
| **推荐** | B. 纯文字题 |
| **理由** | 1. 降低技术复杂度，无需代码沙箱安全防护<br>2. 扩大受众范围（非程序员也可使用）<br>3. MVP快速上线验证 |

### DR-02：选择兼容OpenAI格式而非单一服务商

| 维度 | 内容 |
|------|------|
| **背景** | AI服务商选择 |
| **选项** | A. OpenAI官方API<br>B. 国内大模型（通义千问/文心一言）<br>C. 兼容OpenAI格式的第三方服务 |
| **推荐** | C. 兼容OpenAI格式 |
| **理由** | 1. 接口标准化，切换服务商成本低<br>2. 可灵活选择DeepSeek、智谱AI等国内服务<br>3. 避免单一服务商依赖风险 |

### DR-03：选择五态流转而非三态

| 维度 | 内容 |
|------|------|
| **背景** | 题目状态流转设计 |
| **选项** | A. 两态（已答/未答）<br>B. 三态（未答/已答/已掌握）<br>C. 五态（未答/进行中/已答/需复习/已掌握） |
| **推荐** | C. 五态流转 |
| **理由** | 1. 精细化管理用户学习进度<br>2. 支持错题复习闭环<br>3. 区分"已答但需复习"与"已掌握" |

## 11. 验收标准（Gherkin格式）

### AC-01：用户登录功能

```gherkin
Feature: 用户登录
  Scenario: 用户使用邮箱密码登录成功
    Given 用户已在注册页面填写有效邮箱和密码
    When 用户点击"登录"按钮
    Then 系统验证邮箱和密码正确
    And 系统返回用户信息和JWT Token
    And 用户跳转到首页

  Scenario: 用户登录失败（密码错误）
    Given 用户已在登录页面填写有效邮箱和错误密码
    When 用户点击"登录"按钮
    Then 系统验证密码错误
    And 系统显示"密码错误，请重试"
    And 用户停留在登录页面
```

### AC-02：题目答题功能

```gherkin
Feature: 题目答题
  Scenario: 用户答题正确
    Given 用户已登录且在题目详情页
    When 用户选择正确答案并点击"提交"按钮
    Then 系统判断答案正确
    And 系统显示"回答正确"
    And 系统更新题目状态为"已答"
    And 系统显示AI分析入口

  Scenario: 用户答题错误
    Given 用户已登录且在题目详情页
    When 用户选择错误答案并点击"提交"按钮
    Then 系统判断答案错误
    And 系统显示"回答错误"
    And 系统更新题目状态为"需复习"
    And 系统将题目加入错题本
```

### AC-03：AI题目分析功能

```gherkin
Feature: AI题目分析
  Scenario: AI分析成功
    Given 用户已答题且AI次数未耗尽
    When 用户点击"AI分析"按钮
    Then 系统调用AI API
    And 系统在10秒内返回AI分析结果
    And 系统显示题目分析、解题思路、知识点
    And 系统更新用户AI已用次数

  Scenario: AI分析超时
    Given 用户已答题且AI次数未耗尽
    When 用户点击"AI分析"按钮
    And AI API响应超过10秒
    Then 系统终止请求
    And 系统显示"AI服务繁忙，请稍后重试"
```

Feature: 题库管理

    Given 管理员已登录管理后台
    When 管理员上传符合格式的Excel文件

    And 系统批量插入题目数据
