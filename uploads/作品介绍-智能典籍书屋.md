# 智能典籍书屋 — AI+Web 全栈开发项目

## 1. 项目详情及使用方式

### 项目简介

**智能典籍书屋** 是一个基于 **Vue 3 + Spring Boot** 的智能图书馆管理系统，专为 AI+Web 竞赛开发，核心特色是深度集成 AI 能力，为传统图书馆赋能。

项目支持两类用户角色：
- **读者**：浏览馆藏、借阅图书、预约图书、使用 AI 智能问答、获取个性化推荐、上传自定义头像、管理个人借阅历史
- **管理员**：图书管理、分类管理、用户管理、借阅管理、预约管理、数据统计分析

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3 + Composition API + Element Plus + Pinia + Vue Router + GSAP 动画 + tsParticles |
| 后端 | Spring Boot 3 + MyBatis-Plus + MySQL 8 + JWT + Lombok |
| AI | 统一 AI 客户端封装，支持多模型切换，智能问答/个性化推荐/AI 典籍精读 |

### 本地运行方式

**前端启动：**
```bash
cd library-management
npm install
npm run dev
```
前端默认运行在 `http://localhost:5173`

**后端启动：**
```bash
cd library-management-backend
mvn clean install
mvn spring-boot:run
```
后端默认运行在 `http://localhost:8080`，context-path = `/api`

**数据库初始化：**
```sql
CREATE DATABASE library_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE library_db;
SOURCE library_db.sql;
```
测试账号：
- 管理员：`admin` / `admin123`
- 普通用户：`user001` / `123456`

**配置说明：**
- 修改 `application.yml` 中的数据库连接信息
- 修改 `upload.root` 为你本地的图片上传目录
- 默认上传目录：`library-management/public/images/`

## 3. AI 测试技术落地使用情况概述

项目中的 AI 能力采用**后端统一封装**的架构：

### AI 架构设计
- 统一 `AiClient` 接口，支持多模型切换（OpenAI/Anthropic/火山方舟/本地模型）
- 配置化管理，不用改代码就能切换模型
- 多级降级策略，模型超时/限流自动降级到备选模型
- 前端轻逻辑，所有 AI 推理放在后端，保护 API Key

### 三个 AI 应用场景

| 场景 | 落地方式 |
|------|----------|
| **AI 智能问答** | 用户在前端输入问题，后端调用 AI 接口返回回答，流式输出到前端 |
| **个性化推荐** | 根据用户借阅历史，prompt 工程整合用户借阅记录，AI 生成推荐列表 |
| **AI 典籍精读** | 用户上传书页图片或输入章节文本，后端调用 AI 生成内容摘要、核心观点、赏析解读 |

### 配置方式

在 `application.yml` 中配置：
```yaml
ai:
  enabled: true
  model: "gpt-3.5-turbo"
  api-key: "your-api-key"
  base-url: "https://api.openai.com/v1"
```

## 4. 项目中涉及的关键技术

### 前端关键技术

- **Vue 3 Composition API** - 更好的逻辑复用和代码组织
- **GSAP + ScrollTrigger** - 专业级网页动画，滚动触发动画
- **Element Plus** - 优秀的 Vue 3 组件库，开箱即用
- **Pinia** - 简单易用的状态管理，替换 Vuex
- **Vite** - 极速开发体验，热更新秒级刷新

### 后端关键技术

- **Spring Boot 3** - 现代化 Java 开发，自动配置
- **MyBatis-Plus** - 强大的 CRUD 封装，逻辑删除支持
- **JWT** - 无状态身份认证
- **跨域配置** - Spring CORS 跨域支持
- **静态资源映射** - 用户上传头像/图书封面直接通过 Spring Boot 静态资源映射返回

### 设计亮点

- **逻辑删除** - 所有表支持软删除，数据不物理删除
- **自动时间戳** - createTime/updateTime 自动更新
- **角色权限拦截** - 基于注解的角色权限检查
- **文件上传** - 支持头像/封面图片上传，分类存储
- **响应式布局** - 完美适配手机/平板/桌面
