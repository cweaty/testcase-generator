## 测试策略

### 覆盖维度

| 维度 | 说明 |
|------|------|
| 功能正确性 | 正常输入产生正确输出 |
| 参数校验 | 必填缺失、类型错误、边界值、非法值 |
| 认证鉴权 | Token 缺失/无效/过期、角色越权 |
| 业务规则 | 重复操作、状态约束、库存限制 |
| 数据完整性 | 创建→查询一致、更新→查询一致、删除→查不到 |
| 错误处理 | 预期错误码和错误信息 |

### 用例优先级

| 级别 | 符号 | 说明 |
|------|------|------|
| P0 | ★★★ | 核心流程，每次提交必跑 |
| P1 | ★★☆ | 重要功能，每日回归 |
| P2 | ★☆☆ | 补充覆盖，发版前回归 |

### 用例格式

每条用例包含：ID、优先级、场景、请求方式/路径、请求头、请求体/参数、预期响应

## 认证模块 /auth

### TC-AUTH-001 ★★★ 管理员登录成功

- **场景**：管理员使用正确凭据登录
- **方法**：`POST /api/auth/login`
- **请求体**：
```json
{
  "username": "admin",
  "password": "123456",
  "role": "0"
}
```
- **预期**：`code=200`，`data.token` 非空，`data.user.role=0`

### TC-AUTH-002 ★★★ 普通用户登录成功

- **场景**：普通用户使用正确凭据登录
- **方法**：`POST /api/auth/login`
- **请求体**：
```json
{
  "username": "reader01",
  "password": "reader123",
  "role": "1"
}
```
- **预期**：`code=200`，`data.token` 非空，`data.user.role=1`

## 跨模块场景测试

### TC-E2E-001 ★★★ 完整用户生命周期（E2E）

- **流程**：
  1. 用户注册 `POST /api/auth/register`
  2. 用户登录 `POST /api/auth/login`（获取 token）
  3. 浏览图书 `GET /api/books`
  4. 查看详情 `GET /api/books/{id}`
  5. 借阅图书 `POST /api/borrow`
  6. 查看借阅记录 `GET /api/borrow/records`
  7. 发表评论 `POST /api/comments`
  8. 点赞评论 `POST /api/comments/{commentId}/like`
  9. 预约图书 `POST /api/reservation`
  10. 归还图书 `PUT /api/borrow/{recordId}/return`
  11. 退出登录 `POST /api/auth/logout`
- **验证**：每一步均返回正确响应

### TC-E2E-002 ★★★ 完整管理员生命周期（E2E）

- **流程**：
  1. 管理员登录 `POST /api/auth/login`
  2. 新增分类 `POST /api/api/category`
  3. 新增图书 `POST /api/books`
  4. 修改图书 `PUT /api/books/{id}`
  5. 获取用户列表 `GET /api/users`
  6. 新增用户 `POST /api/users`
  7. 禁用用户 `PUT /api/users/{id}/status?status=0`
  8. 新增公告 `POST /api/notice`
  9. 新增数字资源 `POST /api/digital-resource`
  10. 查看借阅统计 `GET /api/borrow/admin/stats`
  11. 删除图书 `DELETE /api/books/{id}`
  12. 退出登录 `POST /api/auth/logout`
- **验证**：每一步均返回正确响应

### TC-E2E-003 ★★☆ 图书完整生命周期

- **流程**：
  1. 管理员新增图书 → `availableStock == totalStock`
  2. 用户借阅 → `availableStock -= 1`，生成借阅记录 status=0
  3. 用户归还 → `availableStock += 1`，借阅记录 status=1
  4. 用户预约（库存为0时）→ 预约 status=0
  5. 图书归还后管理员完成预约 → 预约 status=3
  6. 管理员删除图书（未借出状态）
- **验证**：库存数量在各步骤前后一致

### TC-E2E-004 ★★☆ 并发借阅测试

- **场景**：库存=1的图书，两个用户同时借阅
- **预期**：一个成功，一个返回"库存不足"

### TC-E2E-005 ★★☆ Token 过期测试

- **场景**：使用过期/无效 token 访问需认证接口
- **接口列表**：
  - `GET /api/users/profile`
  - `POST /api/borrow`
  - `POST /api/comments`
  - `PUT /api/users/password`
- **预期**：所有接口均返回 `code=401`

### TC-E2E-006 ★★☆ 数据一致性验证

- **场景**：验证借阅链路数据完整性
- **步骤**：
  1. 记录当前图书 `availableStock` 值
  2. 用户借阅
  3. 再次查询图书，`availableStock` 应 = 原值 - 1
  4. 查询借阅记录，应存在对应记录 status=0
  5. 归还图书
  6. 查询图书，`availableStock` 恢复原值
  7. 借阅记录 status=1

### TC-E2E-007 ★☆☆ 评论评分一致性

- **步骤**：
  1. 查询图书当前评分汇总
  2. 发表一个 5 分评论

  4. 计算 `averageRating` 是否与预期一致
